# 部署规范 · Deployment

> 目标环境：阿里云 ECS，自用。核心诉求：部署简单、数据可备份、可平滑迭代。

## 1. 环境

- **生产**：阿里云 ECS（Linux）+ Nginx + Node.js（PM2 或 Docker）+ SQLite 文件。
- **本地**：`npm run dev` 一键起前后端，使用本地 SQLite 与 `uploads/`。

## 2. 配置与密钥

- 所有环境相关配置走环境变量，本地用 `.env`（**不入库**）。
- 提供 `.env.example` 列出所需变量（端口、数据目录、上传目录、可选访问密码等），入库供参考。
- 严禁硬编码或提交任何密钥（宪法红线）。

## 3. 部署流程

```bash
# 1. 拉取代码
git pull origin main

# 2. 安装依赖
npm ci

# 3. 前端构建
npm run build            # 产出静态资源

# 4. 启动/重启后端
pm2 restart flash-card   # 或 docker compose up -d --build
```

## 4. Nginx（反代 + 静态 + HTTPS）

- `/` → 前端静态资源目录。
- `/api` → 反向代理到后端本地端口。
- 配置 HTTPS 证书（如免费证书），强制 https。
- 上传大小限制与静态资源缓存按需配置。

## 5. 数据备份（宪法数据红线 · 必须）

用户卡片不可再生，**上线即须有备份机制**：

- 备份对象：SQLite `.db` 文件 + `uploads/` 图片目录。
- 建议：定时任务（cron）每日打包备份到独立位置/对象存储。
- 任何涉及 schema 迁移的发布，先备份再执行；迁移脚本可回滚。

## 6. 发布检查清单

- [ ] `typecheck` / `lint` / `test` 全绿
- [ ] `.env` 在服务器就位，无密钥入库
- [ ] 数据库已备份
- [ ] 前端已构建为最新
- [ ] 后端进程健康（PM2/Docker 状态正常）
- [ ] Nginx 配置校验通过并 reload
- [ ] 访问验证：记卡 → 复习 → 打卡闭环可用

## 7. 回滚

- 保留上一个可用版本（git tag / 镜像），异常时快速切回。
- 数据层出问题时，用最近备份恢复 `.db` 与 `uploads/`。
