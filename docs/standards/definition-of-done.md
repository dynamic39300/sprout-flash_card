# 完成的定义 · Definition of Done (DoD)

> 一把统一的尺子：满足以下条件，一个任务 / 功能才算"完成"。
> 对应 SDD 的 **Validate（验证）** 阶段，是宪法质量红线的可执行落地。

## 使用方式

- **任务级 DoD**：`specs/NNN-*/tasks.md` 中每个任务勾选完成前，对照「任务级」清单。
- **功能级 DoD**：一个功能合并/交付前，对照「功能级」清单。
- AI 在声明"完成"前，须逐条自检并如实报告未达项。

## 任务级 DoD（每个 task）

- [ ] 实现符合对应 spec 的意图，未擅自扩大范围
- [ ] 通过类型检查（`npm run typecheck`）
- [ ] 通过 Lint（`npm run lint`）
- [ ] 涉及核心逻辑的，已补/更新单元测试且通过（`npm test`）
- [ ] 无遗留调试代码、无 `console.log` 噪声、无 `TODO` 未说明
- [ ] 注释只解释「为什么」，无冗余复述

## 功能级 DoD（每个 feature，合并前）

**规格与需求**
- [ ] 覆盖 spec 的**全部 EARS 验收标准**，逐条核验通过
- [ ] Out of Scope 未被越界实现
- [ ] 需求/行为若有变化，已**先更新 spec** 再改代码

**代码质量**
- [ ] typecheck / lint / test 全绿
- [ ] 核心逻辑（SM-2、打卡统计等）测试覆盖含边界（跨天、时区、空数据等）
- [ ] 符合 `coding-standards.md`

**安全**
- [ ] 通过 `security.md` 的相关检查（输入校验、上传安全、无密钥入库等）

**文档**
- [ ] 相关文档已同步更新（AGENTS.md 命令、Tech Spec、术语等，按需）
- [ ] `tasks.md` 进度/决策已记录

**交付**
- [ ] 提交遵循 Conventional Commits，且关联 spec（`refs specs/NNN-*/spec.md`）
- [ ] 无 `data/`、`uploads/`、`.env` 等被忽略文件混入
- [ ] 移动端与 PC 端关键路径均已自测（本项目双端要求）

## 发布级 DoD（上线阿里云前）

- [ ] 满足 `deployment.md` 的发布检查清单
- [ ] 满足 `security.md` 的发布前安全检查清单
- [ ] 数据库与图片已备份，备份可恢复
- [ ] 核心闭环（记卡 → 复习 → 打卡）线上验证通过

## 原则

- DoD 是**门槛不是形式**：未达项要么补齐，要么显式记录为已知限制并获认可。
- 右尺度：平凡改动（改文案/修小 bug）走任务级即可，不必套完整功能级流程。
