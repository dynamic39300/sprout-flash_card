# 安全规范 · Security Standards

> 本项目为**公网部署 + 图片上传 + 免登录**的自用工具，安全面不可忽视。
> 本规范细化宪法第四条「质量红线」中的数据与密钥要求，AI 与人类协作者均须遵守。

## 1. 威胁模型（自用场景）

| 资产 | 威胁 | 影响 |
|------|------|------|
| 卡片数据（SQLite） | 网址泄露被他人读取/篡改 | 隐私泄露、数据被破坏 |
| 上传图片 | 恶意文件、路径穿越写入 | 服务器被写入危险文件 |
| 服务器 | 注入、依赖漏洞 | 被入侵 |
| 密钥/配置 | 提交入库、日志泄露 | 凭证外泄 |

> MVP 免登录是**已知的、经用户确认的权衡**（见 PRD/宪法）。安全设计据此，并预留访问密码作为最小防护。

## 2. 密钥与配置

- 一切密钥、口令、凭证**只走环境变量**（`.env`，不入库）；`.env.example` 仅作占位说明，严禁填真实值。
- 严禁将密钥硬编码进源码、日志、错误信息、提交历史。
- 提交前自查 diff，杜绝 `.env`、`*.db`、`uploads/` 等敏感/数据文件入库。

## 3. 输入校验（后端强制）

- **所有**外部输入（请求体、查询参数、路径参数、上传文件）在服务端做 schema 校验；前端校验仅为体验，不可信。
- 白名单优先：类型、长度、范围、枚举值显式限定。
- 数据库访问使用参数化查询 / ORM 绑定参数，**禁止**字符串拼接 SQL，杜绝注入。

## 4. 图片上传安全（重点）

- **类型校验**：按 `ALLOWED_IMAGE_TYPES` 白名单校验 MIME 与文件魔数（magic bytes），不仅信任扩展名与请求头。
- **大小限制**：不超过 `MAX_UPLOAD_MB`，防止磁盘耗尽。
- **文件名安全**：服务端重新生成随机文件名，**禁止**使用用户提供的文件名；拼接路径时规范化并校验，防止 `../` **路径穿越**。
- **存储隔离**：只写入 `UPLOAD_DIR`；该目录不可执行、不解析脚本。
- 返回给前端的是受控的相对路径/URL，不暴露服务器绝对路径。

## 5. Web 安全

- **XSS**：卡片文本渲染做转义/净化；富文本（如有）用受信任的净化库。React 默认转义，避免滥用 `dangerouslySetInnerHTML`。
- **传输**：生产强制 HTTPS（Nginx 层），敏感操作不走明文。
- **响应头**：由 Nginx/后端设置基本安全头（如 `X-Content-Type-Options: nosniff`、合理的 CSP）。
- **CORS**：仅允许受信任来源；自用场景可锁定为同源。
- 错误响应对外不泄露堆栈/路径/版本等内部细节（内部日志可留）。

## 6. 访问控制（预留）

- MVP 免登录。`.env` 的 `ACCESS_PASSWORD` 为**预留的最小防护**：设置后前端需携带口令方可访问。
- 架构上预留「访问校验中间件」挂载点，便于后续无返工地启用（宪法范围纪律）。
- 提醒运营者：免登录期间**勿公开分享网址**。

## 7. 依赖与供应链

- 用 npm 安装并锁定版本（提交 lockfile）。
- 定期 `npm audit`，及时处理高危漏洞；引入新重量级依赖需评审（见 AGENTS.md 边界）。
- 不引入来源不明、无维护的依赖。

## 8. 数据安全与备份

- 卡片与图片不可再生：上线即须有备份（DB 文件 + `uploads/`），见 `deployment.md`。
- 任何删除采用**软删除**（`deleted_at`），避免误删不可恢复。
- schema 迁移前先备份，迁移脚本可回滚。

## 9. AI 协作中的安全底线

- AI 不得执行不可逆破坏性数据操作（删库、无备份迁移），此类操作先问（Ask First）。
- AI 不得提交密钥/数据文件；发现风险主动阻止并提示。
- AI 遵守 Skill 的"无意外原则"（见 `skill-design.md`）。

- [ ] 无密钥/数据文件入库，`.env` 就位于服务器
- [ ] 所有接口输入有服务端校验

- [ ] SQL 全部参数化，无拼接
- [ ] 生产强制 HTTPS，基础安全响应头就位
- [ ] 对外错误信息不泄露内部细节
- [ ] `npm audit` 无未处理高危项
