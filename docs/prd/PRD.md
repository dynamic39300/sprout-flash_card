# 产品需求文档 · PRD

- **产品名称**：flash_card（知识记忆闪卡）
- **版本**：v1.0（MVP）
- **文档状态**：已评审确认（设计决策见文末决策记录）
- **最后更新**：2026-07-04

---

## 1. 背景与问题

日常学习中会积累大量零散知识，但「记下来」不等于「记得住」。传统笔记只沉淀不复习，遗忘曲线导致知识快速流失。用户需要一个能**低摩擦记录 + 按记忆规律复习 + 有正反馈坚持**的个人工具。

## 2. 目标用户与场景

- **用户**：产品作者本人（自用），未来可能扩展给他人。
- **典型场景**：
  - 学到一个知识点，随手（手机/PC）记成一张问答卡。
  - 每天打开应用，系统给出「今日待复习」卡片，逐张回忆并自评。
  - 通过热力图看到自己的坚持轨迹，获得持续动力。

## 3. 产品目标（价值主张）

> Anki 的记忆算法 + Flomo 的轻记录 + 打卡类应用的坚持激励。

核心价值优先级（继承宪法第一条）：记忆效果 > 顺滑体验 > 坚持激励 > 其他。

## 4. 功能范围（MVP）

### 4.1 功能一：记录知识闪卡

**描述**：用户创建正反面问答卡（Q/A 卡），正反面均支持文本与图片。

**详细需求**：
- 卡片结构为**正面（问题/提示）+ 背面（答案）**。
- 正反面均支持：纯文本输入；图片**粘贴**（截图 Ctrl/Cmd+V）与**上传**（相册/文件）。
- 支持为卡片添加标签（tags），便于筛选。
- 手机端与 PC 端均可创建、编辑、删除卡片，数据共用一份。

**验收标准（EARS）**：
- WHEN 用户在正面或背面输入文本并保存，THE SYSTEM SHALL 持久化该卡片并使其出现在卡片库。
- WHEN 用户在 PC 端粘贴（Ctrl/Cmd+V）一张截图到编辑区，THE SYSTEM SHALL 将图片作为该面素材保存。
- WHEN 用户在手机端选择相册/拍照上传图片，THE SYSTEM SHALL 将图片作为该面素材保存。
- WHEN 用户在任一设备创建卡片，THE SYSTEM SHALL 使其在另一设备刷新后可见（同一数据源）。
- WHERE 卡片被删除，THE SYSTEM SHALL 以可控/可恢复的方式处理，不得静默永久丢失（见宪法数据红线）。

### 4.2 功能二：按记忆规律推送每日复习

**描述**：系统根据间隔重复算法（SM-2）计算每张卡的下次复习日期，用户每天回顾当日到期卡片。

**详细需求**：
- 采用 **SM-2** 算法调度。新卡创建后进入调度，次日起可被安排复习。
- 推送方式为**应用内**：打开应用即在首页看到「今日待复习」列表/卡片流。
- 复习交互：先看正面 → 用户回忆 → 点击翻到背面 → 自评（Again / Hard / Good / Easy）→ 系统据评分更新下次到期日 → 下一张。
- 全部完成后显示「今日已清空」状态。

**验收标准（EARS）**：
- WHEN 用户打开首页，THE SYSTEM SHALL 展示所有到期（due_date ≤ 今天）的卡片作为今日复习队列。
- WHEN 用户对一张卡评分为 Again，THE SYSTEM SHALL 缩短其间隔使其很快重新出现。
- WHEN 用户对一张卡评分为 Good/Easy，THE SYSTEM SHALL 按 SM-2 增大间隔并更新 due_date。
- WHEN 今日复习队列全部完成，THE SYSTEM SHALL 显示「今日已清空」反馈。
- IF 某天没有到期卡片，THEN THE SYSTEM SHALL 显示空状态而非报错。

### 4.3 功能三：坚持打卡（佛系激励）

**描述**：记录用户的复习坚持轨迹，用累计天数与热力图给予正反馈。**不做连胜清零的硬约束**。

**详细需求**：
- 记录每天的复习行为，按日聚合。
- **热力图**（GitHub 风格）：每天复习越多，格子颜色越深。
- **累计复习天数**：展示「已坚持复习 XX 天」。
- **今日进度**：今天已复习 X / Y 张，完成时给予轻量正反馈（如进度环）。
- 明确：不强调「连胜中断清零」，避免打卡焦虑。

**验收标准（EARS）**：
- WHEN 用户当天完成至少一次复习，THE SYSTEM SHALL 在热力图上标记当天并计入累计天数。
- WHEN 用户查看坚持页面，THE SYSTEM SHALL 展示热力图、累计复习天数与今日进度。
- WHILE 用户当天尚未复习完全部到期卡，THE SYSTEM SHALL 展示今日进度 X/Y。

## 5. 非功能需求

- **平台**：移动端优先的响应式网页，PC 亦完整可用；可选 PWA「添加到主屏幕」。
- **多设备**：手机/PC 访问同一阿里云服务器，共用一份数据，实时一致。
- **登录**：MVP **免登录**（知道网址即可用）；架构预留后续加访问密码的能力。
- **性能**：自用量级；常见操作响应 < 300ms（本地网络下）。
- **数据安全**：卡片数据珍贵不可再生，需可备份（DB 单文件 + 图片目录）。

## 6. Out of Scope（本期不做）

- 账号体系、多用户、权限与分享。
- 外部渠道推送（邮件/微信/系统通知）——本期仅应用内。
- 连胜清零/补签卡等硬核激励机制——本期佛系。
- 单面笔记卡形态——本期仅正反面问答卡。
- 云端图片存储（OSS）、跨端离线同步冲突处理——架构预留，不实现。
- FSRS 等高级算法——本期用 SM-2，后续可升级。

## 7. 里程碑

1. **M1 核心闭环**：记卡（文本+图片）→ 今日复习（SM-2）→ 坚持热力图，本地可跑。
2. **M2 打磨**：移动端交互细节、卡片库筛选/搜索、空状态与反馈。
3. **M3 部署**：阿里云 ECS + Nginx 上线自用，数据备份方案。

## 8. 设计决策记录（Decisions）

| 决策点 | 结论 | 备注 |
|--------|------|------|
| 产品形态 | 移动端优先在线网页，部署阿里云自用 | 手机 + PC 共用数据 |
| 卡片形态 | 正反面问答卡（Q/A） | 主动回忆效果最佳 |
| 复习推送 | 应用内「今日待复习」 | 本期不做外部推送 |
| 打卡激励 | 佛系（累计天数 + 热力图） | 不做连胜清零 |
| 登录 | 免登录 | 架构预留访问密码 |
| 记忆算法 | SM-2 | 后续可升级 FSRS |
