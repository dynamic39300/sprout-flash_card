优化点内容：

> 当前的页面UI设计，太难看了——我的直观感受，但我也说不上来非常具体的难看点；你有没有审视ui设计规范、美感，审配的评判方法，我们对当前的UI、交互设计进行自我审视。

优化的思路:

## step1:熟悉当前设计

1️⃣ 设计规范，
2️⃣ 设计令牌
3️⃣ 实际样式/组件

1. 设计规范（Design System / 设计原则文档）
是什么：文字化的设计原则、方向、决策依据——回答"为什么这样设计""设计意图是什么"，本身不是代码，是给人（和 AI）看的指导文档。
对应文件：docs/design/design-system.md，例如里面写的：设计原则，
1. **一屏一事**：不在同一视口堆品牌、eyebrow、大标题、进度与装饰。
2. **摩擦分层**：记录 → 趋近零；复习 → 保留主动回想一步；打卡 → 零（自动记日志）。
3. **内容优先**：正文字号/行距 > 装饰；标签、due date 用 mono 小字降噪。
4. **会话沉浸**：复习进入 `session` 模式——收起 Brand / eyebrow / 常规页眉；底栏弱化。

这层只定“原则”，不定具体色值、字号数字（虽然也列了色板参考值，但重点是“意图”）

2.设计令牌（Design tokens）
是什么：把设计规范里的抽象表决，落成一份可被代码直接应用的变量表——颜色
