# 前端设计体系 · Design System

> 依据 `frontend-design` skill：做有主见、非模板化的设计，避免 AI 默认三套路（奶油底+高对比衬线+陶土色 / 近黑底+荧光单色 / 报纸细线排版）。
> 把大胆押在**一个签名元素**上，其余保持克制。所有颜色/字体决策都从本文件派生。

## 设计简报（Brief）

- **主体**：flash_card —— 个人知识记忆闪卡工具。
- **受众**：作者本人（开发者 / 知识工作者）。
- **页面唯一职责**：让「记录知识」像一次安静、有质感的仪式，让「复习」有揭晓的成就感。

## 设计方向：Riso Study（孔版印刷学习台）

灵感来自 **risograph（孔版印刷）** 美学——学习手账 / 独立出版物常用：有限而鲜亮的油墨色、可见的**分层与轻微错位（misregistration）**、纸张颗粒质感。它扎根于主体世界（知识卡片 = 印刷的学习材料；油墨分层 = 记忆在时间中层层夯实），且明确避开三种默认套路。

**一处大胆（可辩护的风险）**：用**荧光粉**作为唯一的高能强调色，配合**错位描边 + 颗粒质感**。理由：让工具有"亲手印刷的学习材料"的触感，独特且难忘；而结构色（蓝）、克制留白与清晰排版保持全局纪律——boldness 只花在签名元素上。

## 色板（Color Tokens）

主题色为**绿色系**，承载「成长 / 希望 / 不断积累」的知识学习隐喻。

| Token | Hex | 用途 |
|-------|-----|------|
| `--paper` | `#E7EADF` | 页面底色（微微带绿的纸，非奶油色） |
| `--paper-raised` | `#F3F6EA` | 卡片/浮起面 |
| `--ink` | `#15201A` | 主文本、结构线（墨绿黑） |
| `--ink-soft` | `#57645B` | 次要文本 |
| `--grow` | `#0F9A5A` | **签名主题色·成长绿**：主行动、揭晓、选中态、正反馈 |
| `--grow-bright` | `#21B06A` | 亮绿：卡片错位上层（新生） |
| `--pine` | `#0A5A3C` | 深松绿：结构色（导航/边框/标题重点）与错位下层（积累） |
| `--danger` | `#D4503A` | **功能性例外**：仅用于删除/错误，保证一眼可辨 |

> 规则：成长绿是主题主色，用于「主要动作 + 揭晓 + 选中 + 正反馈」；深松绿承担结构与"沉稳的积累"层；红橙仅用于破坏性/错误，避免与绿色语义混淆。
> 一处大胆保持不变（错位 + 颗粒），只是把双层错位改为**亮绿/深绿**——两层绿正好隐喻知识一层层生长、累积。

## 字体（Typography）

内容以**中文**为主，字体栈必须中文优先、拉丁/数字取用有个性的显示体。

| 角色 | 字体栈 | 说明 |
|------|--------|------|
| Display 显示 | `"Bricolage Grotesque", "PingFang SC", "Noto Sans SC", sans-serif` | 有个性的现代 grotesque，用于标题、卡片正面，克制使用 |
| Body 正文 | `"PingFang SC", "Noto Sans SC", "Inter", system-ui, sans-serif` | 中文阅读舒适、可靠 |
| Utility 数据 | `"Space Mono", ui-monospace, monospace` | 数字/日期/标签/统计的"数据感" |

- 类型比例（移动优先）：`12 / 14 / 16 / 20 / 28 / 40`（rem 基准 16）。
- 字重：Display 用 600–700 且加负字距；正文 400–500；数据用等宽 400。

## 布局（Layout）

- **移动优先**、单列、拇指友好；最大内容宽约 `540px` 居中，PC 上留出安静的两侧空白。
- 底部固定导航（4 项：今日复习 / 记卡片 / 卡片库 / 我的坚持），触控目标 ≥ 44px。
- 间距节奏用 8pt 网格（4/8/12/16/24/32/48）。
- 圆角克制：卡片 `14px`，控件 `10px`（非零圆角，与报纸套路区隔）。

```
移动布局（ASCII 线框）
┌───────────────────────┐
│  eyebrow · 记卡片       │  ← 小标签(mono)+display 标题
│                        │
│   ┌─────────────────┐  │
│   │  ▓ 正面 (FRONT)  │  │  ← 卡片本体=主角
│   │  [文本…]         │  │     riso 错位描边(亮绿/深绿)
│   │  [图片缩略]      │  │     纸张颗粒质感
│   └─────────────────┘  │
│   ┌─────────────────┐  │
│   │  ▓ 背面 (BACK)   │  │
│   └─────────────────┘  │
│   # 标签  # 标签  +     │
│                        │
│   [ 保存卡片 ]  ← 成长绿 │

└───────────────────────┘
```

## 签名元素（Signature）

**"孔版卡片"**：卡片本体渲染为带 **riso 错位** 的印刷卡——在卡片边缘用亮绿/深绿各偏移 2–3px 的硬描边（模拟套色不准），叠加细颗粒纹理。
- 记卡片页：正反面就是两张可填写的印刷卡（正面亮绿偏移、背面深松绿偏移），主角地位；两层绿隐喻知识层层生长。
- 复习页（功能二）：卡片**翻面揭晓**答案时，错位从收拢到展开，形成"揭晓"的记忆时刻。

## 质量地板（不喧哗地达标）

- 响应式下探到手机；键盘焦点可见；`prefers-reduced-motion` 时关闭错位动画与过渡。
- 颗粒质感用极低透明度 SVG/`background`，不干扰阅读。
- 对比度满足可读性（正文 `--ink` on `--paper`）。

## 文案（Voice）

- 站在用户一侧、用主动语气、动词直白：按钮写「保存卡片」，成功提示写「已保存」。
- 空状态是行动邀请，而非情绪化文案：如卡片库空时——「还没有卡片。记下第一个知识点吧。」
- 错误信息说清"发生了什么 + 怎么办"，用界面口吻，不道歉、不含糊。

## 落地方式

- 令牌落到 `client/src/styles/tokens.css`（CSS 自定义属性），全局样式与组件从令牌派生，禁止散落硬编码颜色。
- 字体用 Google Fonts（Bricolage Grotesque / Space Mono）+ 系统中文字体，无网络时优雅降级到系统字体。
