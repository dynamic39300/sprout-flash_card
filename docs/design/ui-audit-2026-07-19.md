# UI 审视报告 · 2026-07-19

> 对青芽 Sprout 当前实现（specs 001–004 已落地）进行系统性视觉/交互自审。
> 方法论：6 维度客观评判 + 竞品对标 + 令牌/代码核验。
>
> **状态更新（2026-07-19 下午）**：本报告 P0/P1/P2 全部改造点已实施，详见 `specs/004-ui-polish/plan-visual-refresh.md` 的落地记录与 `specs/004-ui-polish/spec.md` Phase 2 验收标准。

---

## 0. 评判方法论

| 维度 | 核心问题 | 客观检验 |
|------|----------|----------|
| **层次/景深** | 眼睛能分出前景与背景吗 | 截图模糊后是否可辨区块；底色与卡面明度差 ≥ 5% |
| **对比/可读** | 文字和分隔够清晰吗 | 正文对比度 ≥ 4.5:1；次要文本 ≥ 3:1 |
| **节奏/对齐** | 间距成体系、左缘对齐吗 | 叠 8pt 网格验证；间距值 ≤ 7 档 |
| **色彩克制** | 颜色是否只为"意义"而用 | 一屏色相 ≤ 2；强调色只给主行动 |
| **字体秩序** | 字号/字重/字体族是阶梯还是杂货铺 | 一屏"字号×字重×族"组合 ≤ 6 |
| **细节一致性** | 圆角、描边、图标是否同一套视觉语言 | 圆角值种类 ≤ 3；描边粗细统一；图标同源 |

优秀冷静界面公式：**高层次感 + 低色彩噪声 + 严格字体/间距秩序 = 高级感**。

---

## 1. 诊断结果

### 1.1 🔴 景深几乎为零（最主要元凶）

**现象**：页面底色 `--paper: #e7eadf` 与卡面 `--paper-raised: #f3f6ea` 明度差仅约 4%；全局无阴影 token；卡片只靠 14% 透明度的发丝线分隔。

**后果**：卡片"浮不起来"，整页粘成一片灰绿色块，眯眼无法区分区块层级。

**竞品对照**：flomo 用白卡 + 极浅灰底（明度差 ~12%）+ 轻影；Capacities 更是 pure white 卡片悬浮于灰底。

**修法**：
- 卡面提亮至接近白（如 `#fbfdf7` 或 `#fff`）
- 底色可保持或略沉（如 `#e3e6dc`）
- 新增 `--shadow-sm` / `--shadow-md` token
- 卡片用柔影取代硬描边

### 1.2 🔴 深色硬描边 + Riso 错位常驻（视觉噪声源）

**现象**：按钮 `1.5px solid ink`、复习卡 `1.5px ink` + 3px 错位伪元素常驻，评分按钮同样描边。全局几乎所有可交互元素都有深色轮廓。

**后果**：重描边 = "线框图/蓝图"风格，与设计系统宣称的"冷静、flomo 式"完全相悖；Riso 错位静态呈现像"印刷错位 bug"，无法传达设计意图。

**竞品对照**：flomo 无描边，用阴影/留白分层；Skola 只用极浅 1px；Anki 新版也已放弃硬线框。

**修法**：
- 卡片/面板：去掉或减淡到 `hairline` 级别；改用阴影表达浮起
- 按钮：primary 去描边（纯色块 + 柔角）；secondary 用浅灰描边
- Riso 错位：**仅在揭晓动效那一刻**短暂出现（0.3s 后 fade），不作为静态装饰

### 1.3 🟠 等宽大写微标签过度泛滥

**现象**：`Space Mono` + `uppercase` + `letter-spacing: 0.1em` 出现在 ≥8 处（面标签、字段标签、tag、元信息、导航文字、热力图标头、计数等）。

**后果**：
- 满屏 10–11px 大写等宽 = 终端/仪表盘感，与"生长/温暖"气质冲突
- 等宽字体无法正确渲染中文，导致中英混排字重不齐
- 同一视觉角色（"标签"）出现频率过高本身就是层次噪声

**竞品对照**：flomo 标签用正文体小号 + 浅灰；Capacities 标签同样是正文体。只有 GitHub 贡献墙用 mono，那是因为数据仪表盘语境。

**修法**：
- mono 严格收敛到：日期、张数、热力图数值、快捷键 hint
- 所有含中文的标签回归 `--font-body`，正常大小写
- 减少标签出现频率，合并同义标签

### 1.4 🟠 导航图标 = 杂牌 Unicode 符号

**现象**：底栏用 `◑ ✎ ▤ ✦` 四个来自不同符号族的 Unicode 字符做图标。

**后果**：
- 笔画粗细、视觉重量、风格完全不统一
- 跨平台/字体渲染不一致（有的变彩色 emoji，有的极细）
- 一眼"临时凑合"感

**修法**：换为同一套线性 SVG 图标（24px 画板、1.5–2px 描边），风格与青芽 logo 一致。

### 1.5 🟡 主色偏"机构绿" + 强调色过度分散

**现象**：`--grow: #0f9a5a` 高饱和正绿同时用于：主按钮、选中态、揭晓、正反馈、到期高亮、热力图、标签激活、面板左边框、进度条……强调色"到处都是 = 哪里都不强调"。

**竞品对照**：flomo 绿只出现在 FAB + 少量 badge；Capacities 品牌紫只给极少的交互焦点。

**修法**：
- 主绿略降饱和偏叶绿（如 `#2d9d6a`），更有机
- 强调色仅用于：主行动按钮、复习揭晓、热力图深色格（≤3 处同屏）
- 面板左边框、标签激活等改用中性灰/浅绿

### 1.6 🟡 规范/实现偏差

| 规范要求 | 实际实现 | 偏差 |
|----------|----------|------|
| 累计天数"克制字号，不以巨型英雄数字抢戏" | `40px / weight 800` | 字号过大、字重过重 |
| 复习中底栏"半透明弱化" | 底栏整段隐藏 | 过于激进 |
| "轻错位仅挂在卡片本体" | 错位常驻且静态 | 应为动效 |

## 2. 评分汇总

| 维度 | 当前评分 (1–5) | 主因 |
|------|---------------|------|
| 层次/景深 | 2 | 无阴影、底/卡面明度差不足 |
| 对比/可读 | 3 | 正文尚可，但次要文本灰度偏淡 |
| 节奏/对齐 | 3.5 | 8pt 网格基本遵守，间距档位控制合理 |
| 色彩克制 | 2.5 | 强调色过散，色相单一但"糊"了 |
| 字体秩序 | 2.5 | mono/大写滥用，组合 >8 种 |
| 细节一致性 | 2 | 图标杂牌，描边/阴影混用，圆角尚可 |

**综合**：2.6 / 5 — 骨架合理，但"表面处理"大量拉低质感。好消息是，改造不需动业务/布局，只动令牌和浅层样式即可大幅提升。

## 3. 改造优先级

按"投入小 × 观感提升大"排序：

| 优先级 | 改造项 | 预期效果 |
|--------|--------|----------|
| P0 | 建立景深（提亮卡面 + 柔影 token + 去硬描边） | 解决 60% 的"灰扑扑" |
| P0 | Riso 错位改为动效非静态 | 消除"印歪了"违和 |
| P1 | 收敛 mono/大写到数据场景 | 回归温暖、减噪 |
| P1 | 底栏图标换同一套 SVG | 消除廉价感 |
| P2 | 主绿调色 + 强调色收敛 | 提升高级感 |
| P2 | 坚持页数字克制 + 底栏弱化改为半透明 | 对齐规范 |
