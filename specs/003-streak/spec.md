# [003] 坚持打卡（佛系激励）· Spec

- **功能 ID**：003-streak
- **状态**：Approved
- **关联 PRD**：`docs/prd/PRD.md` 4.3
- **最后更新**：2026-07-04

## 1. 目标（Why）

用轻量的正反馈让用户感知到自己的积累——「我已经坚持复习了 X 天」的成就感，比清零连胜的压力更适合长期维持。热力图让沉默的积累变得可见。

## 2. 用户场景

- 打开「坚持」tab → 看到热力图（近 16 周）+ 累计复习天数 + 今日进度。
- 某天忘了复习也不会被"警告"，格子只是留白，不影响累计。
- 复习越多的天，格子越深绿；一眼看出哪段时间最活跃。

## 3. 功能需求（What）

- **热力图**：展示过去 16 周（112 天）每日复习张数，颜色 4 档（0 / 1-3 / 4-9 / 10+）。
- **累计复习天数**：`review_logs` 中 `review_day` 的不重复天数。
- **今日进度**：今天已复习张数 / 今日到期总张数，展示进度环或进度条。
- **鼓励文案**：根据累计天数显示阶段性文案（首次、3天、7天、30天…），佛系口吻。

## 4. 验收标准（EARS）

- WHEN 用户当天完成至少一次复习，THE SYSTEM SHALL 在热力图上标记当天并计入累计天数。
- WHEN 用户查看坚持页面，THE SYSTEM SHALL 展示热力图、累计复习天数与今日进度。
- WHILE 用户当天尚未复习完全部到期卡，THE SYSTEM SHALL 展示今日进度 X/Y。
- IF 用户从未复习，THE SYSTEM SHALL 展示空热力图与「第一次复习就是开始」提示。

## 5. Out of Scope

- 连胜（streak）清零与补签机制。
- 分享、导出热力图。
- 周/月聚合统计。

## 6. 依赖

- 依赖功能二：`review_logs` 表已存在并在评分时写入数据。
