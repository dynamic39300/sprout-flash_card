# [001] 记录知识闪卡 · Spec（规格）

- **功能 ID**：001-card-capture
- **状态**：Approved
- **关联 PRD**：`docs/prd/PRD.md` 4.1
- **最后更新**：2026-07-04

## 1. 目标（Why）

让用户能低摩擦地把知识记录成正反面问答卡（文本 + 图片），并在卡片库中查看/管理。这是整个记忆闭环的输入端。

## 2. 用户场景

- 作为用户，我想快速把一个知识点记成「正面=问题、背面=答案」的卡片，正反面都能配图，以便日后复习。
- 我在手机上随手记，也在 PC 上粘贴截图整理，数据是同一份。

## 3. 功能需求（What）

- 创建卡片：正面、背面各含文本；正反面均可添加图片（PC 粘贴 Ctrl/Cmd+V、移动/桌面上传）。
- 卡片可加标签（tags）。
- 卡片库：列表展示已创建卡片，可查看正反面与图片，支持按标签/关键词筛选。
- 编辑与删除卡片（删除为软删除，可恢复）。

## 4. 验收标准（EARS）

- WHEN 用户填写正面或背面文本并保存, THE SYSTEM SHALL 持久化该卡片并使其出现在卡片库。
- WHEN 用户在编辑区粘贴（Ctrl/Cmd+V）一张图片, THE SYSTEM SHALL 上传并将其作为当前面的素材附加到卡片。
- WHEN 用户通过文件选择上传图片, THE SYSTEM SHALL 同样附加为当前面素材。
- WHEN 用户为卡片添加标签并保存, THE SYSTEM SHALL 保存标签并支持按其筛选。
- WHEN 用户在卡片库输入关键词, THE SYSTEM SHALL 返回正/背面文本匹配的卡片。
- WHEN 用户删除卡片, THE SYSTEM SHALL 软删除（保留数据、从库中隐藏），不永久丢失。
- IF 正反面文本与图片均为空, THEN THE SYSTEM SHALL 阻止保存并提示至少填写一面。
- IF 图片超过大小上限或类型不被允许, THEN THE SYSTEM SHALL 拒绝并给出清晰提示。

## 5. Out of Scope（本功能不做）

- 复习调度与 SM-2（功能二）、打卡热力图（功能三）。
- 富文本/Markdown 渲染（本期纯文本 + 图片）。
- 卡片间关联、批量导入、云端图片存储。

## 6. 依赖与假设

- 依赖后端 SQLite 与本地图片存储（见 Tech Spec）。
- 免登录，单用户，一份数据。
