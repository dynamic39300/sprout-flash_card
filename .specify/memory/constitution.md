# 项目宪法 · Constitution

> 本文件定义 flash_card 项目**不可变的最高原则**。它优先于一切其他文档、规范和临时指令。
> 任何 spec、plan、代码实现如与本宪法冲突，一律以宪法为准。修改宪法需显式评审并记录变更。

- **版本**：1.0.0
- **生效日期**：2026-07-04
- **适用对象**：所有人类协作者与 AI 编码助手（Cursor / Claude / Copilot 等）

---

## 第一条 · 产品使命（Why）

flash_card 是一个**个人知识记忆工具**，核心使命是让「记录知识 → 按记忆规律复习 → 长期坚持」这条链路尽可能轻松。

一切功能与技术决策，都应服务于这个使命。当出现取舍时，优先级排序为：

1. **记忆效果**（帮助用户真正记住）
2. **记录/复习的顺滑体验**（低摩擦）
3. **长期坚持的正反馈**（激励）
4. 其他（炫技、扩展性等）

## 第二条 · 规范驱动开发（Spec-Driven Development）

**先规格，后代码。** 本项目采用 SDD 方法论：

1. 任何非平凡功能，必须先有 spec（规格），再有 plan（计划），再有 tasks（拆分），最后才 implement（实现）。
2. 规格是「活文档」与事实来源：**行为变化时，先改规格，再改代码。**
3. 严禁跳过阶段直接从想法跳到编码。评审在每个阶段边界进行。
4. 一个功能 = 一个目录：`specs/NNN-feature-name/`。
5. 验收标准使用 EARS 语法（见 `docs/process/spec-driven-development.md`）。

## 第三条 · 技术栈锁定（Locked Stack）

以下技术选型为项目基线，非经宪法修订不得擅自更换：

- **前端**：React + Vite + TypeScript，移动端优先响应式，PWA 可选
- **后端**：Node.js + TypeScript（单体服务）
- **数据库**：SQLite（单文件，自用量级）
- **图片存储**：服务器本地磁盘（后续可平滑迁移至阿里云 OSS）
- **部署**：阿里云 ECS + Nginx 反代 +（PM2 或 Docker）

> 语言统一 TypeScript（前后端），杜绝无类型 JavaScript 业务代码。

## 第四条 · 质量红线（Non-Negotiable）

以下为不可妥协的底线，AI 与人类都必须遵守：

1. **类型安全**：TypeScript `strict` 模式，禁止无理由的 `any`。
2. **测试**：核心逻辑（间隔重复算法、打卡统计）必须有单元测试，且随实现一起提交。
3. **数据安全**：用户的卡片是不可再生的珍贵数据。任何涉及删除/迁移的操作必须可逆或有备份；数据库文件与图片不入 git。
4. **密钥安全**：任何密钥、凭证严禁硬编码或提交入库，一律走环境变量。
5. **提交规范**：遵循 Conventional Commits，提交信息关联对应 spec。
6. **小步交付**：单次变更聚焦、可评审；一次实现批次控制在 3–5 个任务内，避免上下文漂移。

## 第五条 · AI 协作契约

1. `AGENTS.md`（根目录）是 AI 协作的**单一事实来源**，所有 AI 工具以其为准。
2. AI 在实现前必须阅读相关 spec、plan 与本宪法；不得臆测需求，遇歧义先澄清（Clarify）。
3. AI 必须遵守本仓库全部规范文档；当规范与用户临时指令冲突时，提示冲突并请人类裁决。
4. AI 的产出（代码、文档、提交）需可追溯到对应规格。

## 第六条 · 范围纪律（Scope Discipline）

1. 明确「不做什么」和「做什么」同等重要，每个 spec 必须写「Out of Scope」。
2. 保持轻量：不过度设计、不过早优化。仅在明确带来价值处扩展流程与架构。
3. 自用为先：当前阶段不引入账号体系、多租户、复杂权限（架构预留即可）。
